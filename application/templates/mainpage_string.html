Syntax for probabilistic grammars
The rules for your grammar should be written as a list of production rules in Chomsky normal form:
A -> B C [probability] or
A -> b [probability]
where A,B,C are non-terminals and b is a terminal. Empty productions can't be used. Each rule should start with the left-hand side nonterminal followed by an -> symbol. In expansion rules, a space should be used to delimit separate nonterminals on the left-hand side. The parser will assume that the lefhand side nonterminal of your first rule is the start symbol. Alternate left-hand side expansion can be chained using the | symbol, or entered separately. In each case, the probability for each expansion should be placed in square brackets immediately following the expansion. After the end of each chain of expansions for a rule, a comma delimits the next rule. New rules do not need to be placed onto a new line. As an example, the following grammar shows the syntax structure:


S -> NP VP [0.5]| S PP [0.25]|V NP [0.05] | V PP [0.1] | flies [0.04] | like [0.04] | time [0.02],
VP -> V NP [0.2] | V PP [0.4] | flies [0.16]| like [0.16] | time [0.08],
NP -> Det N [0.4]| N N [0.1]| NP PP [0.1]| arrow [0.16] | flies [0.08] | time [0.16],
PP -> P NP [1.0],
N -> arrow [0.4]| flies [0.2]| time [0.4],
V -> flies [0.4] | like [0.4] | time [0.2],
P -> like [1.0],
Det -> an [1.0]


If a probability is not given for an expansion, then the parser will assume the probability is 1. The parser will allow any probabilities to be entered, but if you expect a proper probability distribution over parses, then the sum of probabilities for expansions of each nonterminal should equal 1.
