File structure for non-probabilistic grammars
The file structure for non-probabilisitc grammars is identical to probabilistic grammars, but the probability field in each rule object is not read.
The rules for you grammar should be written as a list of production rules in chomsky normal form of the form:
A -> B C or
A -> b
where A,B,C are non terminals and b is a terminal. Empty productons can't be used. The JSON file upload should consist of the following fields:
  • The start symbol of your grammar. This should be entered like this: "start_symbol":"S"
  • The nonterminal symbols. This should be entered as a list like this: "non_terminals": ["S","NP","VP","N","V","P","PP","Det"]
  • The terminal symbols. Ths should be entered as a list like this: "terminals": ["time","flies","like","an","arrow"]
  • The rules of the grammar. This should be a list of nested JSONs for each for. As an example the rules should be entered like this:

    "rules": [{"nonterminal":"S","expansion":["NP","VP"],"probability":0.5}, {"nonterminal":"S","expansion":["S","PP"],"probability":0.25}]

    Each rule object should have a "non_terminal" field, an "expansion" field which is a list of the lefthand side elements, even if the rule is terinal]. There is ane example grammar file here.Note this file includes probabilities, but this field is ignored in non-probabilistic mode.
